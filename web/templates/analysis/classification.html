About Classification

The table below presents matching against CAPE sandboxing of similar samples.
The score is computed from the comparison of the graph of API calls observed during the execution of the different samples identified below by their SHA256.

A graph is built for each process using the tuple (thread id, caller address) as node id and the API call name as symbol. As a result, the matching is per process (e.g., if the present sample is executing cmd.exe, it will match highly with any other sample executing the same process).

To compute a numeric matching score, the API call graph is exploded into sub-graphs. Let SG(A) be the set of subgraphs for process A. The score relies on the Jaccard index: size(SG(A) ∩ SG(B)) / size(SG(A) ∪ SG(B)).

Note: Family names are indicative only and extracted from CAPE sandboxing references, which may be subject to error. Always consider the context of discriminative processes.
